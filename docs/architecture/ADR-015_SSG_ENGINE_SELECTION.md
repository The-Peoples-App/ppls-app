# ADR-015: SSG Engine Selection (Markdown-to-Native)

## Status 
Proposed (Epic: Infra/DX)

## Context
To meet the 6-8 week "Bronze" timeline, we need an SSG solution that minimizes "Parsing Overhead" at runtime and maximizes "Staff Ease-of-Use."

## Decision
We will utilize **Front-matter + react-native-markdown-display** for the Pilot.
1. **Source:** `raw_content/*.md` files.
2. **Build Pipeline:** `bin/generate-guide.js` will aggregate these into a `bundled-assets.json`.
3. **Resilience & Optimization (Traceability to ADR-004):**
   - **Binary Size:** By pre-processing Markdown into a single optimized JSON bundle at build-time, we eliminate the need for heavy runtime parsers and multiple file-system hits, keeping the app lightweight for users on limited data plans.
   - **Offline Sovereignty:** The `bundled-assets.json` is packaged directly within the application binary. This ensures the "Static Reader" and "Reward Calculator" require zero network connectivity to function, preventing the app from becoming a "brick" in LA's data deserts.
4. **Logic:** This JSON will be indexed by the **AI Semantic Layer** and rendered via the **Static Reader** UI.

## Demonstration
### The Source: `raw_content/calfresh_p37.md`
This uses **YAML Front-matter** (the block between `---`) to give the app "Metadata" (like the page number) while keeping the "Advocacy Text" clean.

```markdown
---
id: "calfresh-how-to"
title: "How Do I Get CalFresh?"
page: 37
section: "CalFresh"
---

# How Do I Get CalFresh?

You can apply for CalFresh in person, by phone, or online. 

### Quick Steps:
1. **Call DPSS:** (866) 613-3777
2. **Apply Online:** [BenefitsCal.com](https://benefitscal.com)

> **Pro-Tip:** State workers suggest having your ID and Rent Receipt ready before you call to cut through the red tape.
```

### Processing
Runs through the parser: `calfresh_p37.md` -->  `bin/generate-guide.js` --> `bundled-guide.json`

### The Output `assets/bundled-guide.json` (high-performance asset)
This is the "Serialized" version generated by your `bin/generate-guide.js` script. It strips out the complexity so the **Component Registry (ADR-017)** and **AI Search (ADR-014)** can read it instantly.

```json
{
    "version": "1.0.4",
    "last_build": "2026-03-12T17:00:00Z",
    "chapters": [
        {
            "id": "calfresh",
            "pages": [
                {
                    "id": "calfresh-how-to",
                    "metadata": {
                        "title": "How Do I Get CalFresh?",
                        "page": 37,
                        "section": "CalFresh"
                    },
                    "content": "# How Do I Get CalFresh?\n\nYou...",
                    "tokens": ["dpss", "apply", "benefitscal", "id", "rent", "p37"]
                }
            ]
        },
        {
            "id": "free-food",
            "pages": [
                { "id": "p41", "content": "..." }
            ]
        }
    ]
}
```

> **For the AI**: The tokens array in the JSON output acts as a "lite" index, making the **Semantic Search** incredibly fast without needing a heavy database query.

> **For the App**: Because it’s a single JSON file, the **Static Reader** can load the entire CalFresh chapter into memory at once, providing **0ms latency** for the user.

> **For the Staff**: They never see the JSON. They only work in the Markdown file, which looks like a simple Word doc to them.

## Consequences
- **Pro:** 0ms latency for page transitions as the entire chapter is memory-resident.
- **Pro:** Total data sovereignty; the app is 100% functional without a backend connection.
- **Con:** Updates to the "Static" content require a new App Store release or an EAS Update (OTA).
